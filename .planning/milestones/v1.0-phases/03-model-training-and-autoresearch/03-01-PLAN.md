season has NaN rolling features. The final rolling_win value in the last week of a

```python
def _build_prior_season_records(df: pd.DataFrame) -> dict[tuple[str, int], float]:
```
- Build a lookup dict mapping (team, season) -> that team's final rolling_win from the PRIOR season (season - 1).
- CAVEAT: rolling_win uses shift(1) + expanding mean, so the value at the last week of a season represents the win rate going INTO that game, not including it. This means the prior-season tiebreaker is stale by one game (e.g., a 16-game record instead of 17). This is a known minor inaccuracy that does not materially affect baseline accuracy — document it in the function docstring.
- Implementation: for each team+season group, find the last row (max week) and get its rolling_win value. Then shift by +1 season to create the mapping: key is (team, season), value is the team's final rolling_win from season-1.
- Concretely: group df by home_team+season (and separately away_team+season), find the max-week row's rolling_win for each group, then store as (team, season+1) -> rolling_win. Merge home and away lookups (they should agree since both come from the same team's games, just seen from different perspectives). Use the home-perspective data: for each season S, collect all unique teams appearing as home_team, get their home_rolling_win at the latest week. Map as (team, S+1) -> that value.
- Actually, simpler approach: iterate over each (season, home_team) group, find the row with the max week, take its home_rolling_win. Store as (home_team, season+1) -> value. Do the same for (season, away_team) with away_rolling_win. Merge the two dicts (values should be consistent since a team's rolling_win is the same stat regardless of home/away perspective). This gives prior-season record for tiebreaking.

```python
def better_record_baseline(df: pd.DataFrame, season: int) -> float:
```
- NOTE: accepts the FULL multi-season DataFrame and a target season. Per user decision (CONTEXT.md): "Ties broken by prior season record. Home-team fallback only when no prior season data exists."
- Step 1: Build prior-season lookup via _build_prior_season_records(df)
- Step 2: Filter df to the target season
- Step 3: Filter to rows where home_win is not NaN (exclude ties)
- Step 4: Filter to rows where home_rolling_win AND away_rolling_win are not NaN (need records to compare)
- Step 5: For each game, determine prediction:
  - If home_rolling_win > away_rolling_win -> predict home win
  - If away_rolling_win > home_rolling_win -> predict away win
  - If tied (home_rolling_win == away_rolling_win) -> tiebreaker:
    - Look up prior-season records: prior_home = lookup.get((home_team, season)), prior_away = lookup.get((away_team, season))
    - If both prior records exist: predict the team with higher prior-season rolling_win
    - If only one exists: predict that team (it has a track record, the other doesn't)
    - If neither exists (e.g., season 2005 which is first in dataset): fall back to predicting home team wins
- Step 6: Return accuracy = (predictions == actuals).mean()

```python
def compute_baselines(df: pd.DataFrame, season: int) -> dict:
```
