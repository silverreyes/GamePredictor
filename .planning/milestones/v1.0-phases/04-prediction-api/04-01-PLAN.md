```sql
-- Predictions: pre-computed game predictions (one per game, no model versioning)
CREATE TABLE IF NOT EXISTS predictions (
    game_id VARCHAR(20) PRIMARY KEY,
    season SMALLINT NOT NULL,
    week SMALLINT NOT NULL,
    game_date DATE,
    home_team VARCHAR(3) NOT NULL,
    away_team VARCHAR(3) NOT NULL,
    predicted_winner VARCHAR(3) NOT NULL,
    confidence REAL NOT NULL,
    confidence_tier VARCHAR(6) NOT NULL,
    model_id INTEGER,
    actual_winner VARCHAR(3),
    correct BOOLEAN
);

CREATE INDEX IF NOT EXISTS idx_predictions_season_week ON predictions (season, week);
CREATE INDEX IF NOT EXISTS idx_predictions_season ON predictions (season);
```

Per CONTEXT.md: PRIMARY KEY is game_id only (no multi-version tracking). `model_id` stores the experiment_id that generated the prediction (for debugging/auditing only — not exposed in the API response). actual_winner and correct columns are NULL until the weekly refresh pipeline fills them when results are ingested (Phase 6). confidence_tier is VARCHAR(6) to hold "high", "medium", or "low".

**NOTE:** confidence_tier is stored in the DB but derived from confidence + config thresholds. If you change the thresholds in config.py, stored tiers become stale until predictions are regenerated. This is acceptable — regenerating predictions already happens on model reload.

```python
"""Prediction generation pipeline.

Generates pre-computed predictions from the trained model and stores them
in the predictions table. Prediction logic lives here (not in api/) per
CONTEXT.md decision: keeping prediction code with model code.

Functions:
    load_best_model: Load XGBoost model from best_model.json artifact.
    get_best_experiment: Parse experiments.jsonl for the best kept experiment.
    detect_current_week: Data-driven current week detection from schedule.
    generate_predictions: Build features, run inference, store predictions.
"""
```

**load_best_model(path="models/artifacts/best_model.json"):**

- `model.load_model(path)`
- Return model
- Raise FileNotFoundError with descriptive message if path doesn't exist

**get_best_experiment(jsonl_path="models/experiments.jsonl"):**
- Open file, iterate JSON lines
- Track the entry with `keep=True` AND highest `val_accuracy_2023`
- **Normalize `model_path`** in the returned dict: `entry["model_path"] = entry.get("model_path", "").replace("\\", "/")` — experiments.jsonl contains Windows backslashes (`models/artifacts\\model_exp005.json`) which break on Linux (the VPS). Do this before returning.
- Return the full dict (experiment_id, timestamp, params, features, val_accuracy_2023, baseline_always_home, baseline_better_record, hypothesis, etc.)
- Return None if no kept experiments exist
- Raise FileNotFoundError if jsonl_path doesn't exist

**detect_current_week(engine):**
- SQL query against the `schedules` table (NOT `raw_schedules` -- the project uses `schedules` as the table name, see sql/init.sql):
```sql
SELECT MIN(week) as current_week
FROM schedules
WHERE season = (SELECT MAX(season) FROM schedules WHERE game_type = 'REG')
  AND game_type = 'REG'
  AND home_score IS NULL
```
- Also query `SELECT MAX(season) FROM schedules WHERE game_type = 'REG'` for current_season
- Use `pd.read_sql(query, engine)`
- If current_week is None (all games played): return None (offseason)
- Otherwise return tuple: `(current_season, current_week)` where both are Python int (not numpy)

**generate_predictions(model, season, week, engine, confidence_tier_fn, model_id=None):**
- This is the core function. It must handle the fact that unplayed games have NULL scores.
- Step 1: Build features for all COMPLETED games in the season to get rolling stats. Use `build_game_features(seasons=[season])` which returns a DataFrame. This works because rolling features use shift(1)+expanding mean -- the rolling stats for each team carry forward from prior games.
- Step 2: Get schedule info for unplayed games in the target week:
  ```sql
  SELECT game_id, season, week, gameday, home_team, away_team, home_rest, away_rest, div_game
  FROM schedules
  WHERE season = {season} AND week = {week} AND game_type = 'REG' AND home_score IS NULL
  ```
- Step 3: For each unplayed game, construct the feature vector. **This is the riskiest part of the pipeline** — extract it into a helper function `_get_team_rolling_features(features_df, team)` so it can be unit-tested independently:

  **`_get_team_rolling_features(features_df, team) -> dict`:**
  - Find all rows where `home_team == team` OR `away_team == team`
  - Take the row with the maximum week (most recent game)
  - **CRITICAL PREFIX FLIP:** If the team appeared as `home_team` in that row, extract `home_rolling_*` columns. If the team appeared as `away_team`, extract `away_rolling_*` columns. Both sets contain that team's stats — the prefix indicates the team's role in that game, not which stats belong to which team.
  - Return a dict with UNPREFIXED keys (e.g., `"rolling_off_epa_per_play"`) so the caller can re-prefix as needed for the target game's home/away slot.
  - If no rows found for the team, raise ValueError (team hasn't played any games yet this season).

  Then for each unplayed game:
  a. `home_feats = _get_team_rolling_features(features_df, home_team)` → re-prefix as `home_rolling_*`
  b. `away_feats = _get_team_rolling_features(features_df, away_team)` → re-prefix as `away_rolling_*`
  c. Combine: home_rest, away_rest, div_game from schedule + home_rolling_* + away_rolling_*
- Step 4: Get feature column names from `model.get_booster().feature_names` to ensure correct column ordering.
- Step 5: Run `model.predict_proba(X)[:, 1]` to get P(home_win) for each game.
- Step 6: For each game, compute:
  - If `home_prob >= 0.5`: predicted_winner = home_team, confidence = home_prob
  - Else: predicted_winner = away_team, confidence = 1.0 - home_prob
  - confidence_tier = confidence_tier_fn(confidence) (passed in from api/config.py)
  - Include `model_id` in each record dict (the experiment_id, for debugging)
- Step 7: Upsert results into predictions table using SQLAlchemy. Use the same upsert pattern as data/ingest.py:
  ```python
  from sqlalchemy.dialects.postgresql import insert as pg_insert
  stmt = pg_insert(predictions_table).values(records)
  upsert = stmt.on_conflict_do_update(
      index_elements=["game_id"],
      set_={col: stmt.excluded[col] for col in update_columns}
  )
  ```
  Update all columns except game_id on conflict.
- Return the list of prediction dicts (for API response after reload).

```python
"""API configuration and settings."""
import os
from dotenv import load_dotenv

load_dotenv()


class Settings:
    """Application settings loaded from environment."""

    RELOAD_TOKEN: str = os.environ.get("RELOAD_TOKEN", "")
    DATABASE_URL: str = os.environ.get("DATABASE_URL", "")
    MODEL_PATH: str = os.environ.get("MODEL_PATH", "models/artifacts/best_model.json")
    EXPERIMENTS_PATH: str = os.environ.get("EXPERIMENTS_PATH", "models/experiments.jsonl")

    # Confidence tier thresholds (configurable, not hardcoded per CONTEXT.md)
    CONFIDENCE_HIGH: float = float(os.environ.get("CONFIDENCE_HIGH", "0.65"))
    CONFIDENCE_MEDIUM: float = float(os.environ.get("CONFIDENCE_MEDIUM", "0.55"))

    # CORS origins for React dashboard (Phase 5)
    CORS_ORIGINS: list[str] = [
        "http://localhost:3000",
        "http://localhost:5173",
    ]


settings = Settings()


def get_confidence_tier(confidence: float) -> str:
    """Map raw confidence score to tier label.

    Thresholds are configurable via Settings. Default:
    - high: >= 0.65
    - medium: >= 0.55
    - low: < 0.55

    Args:
        confidence: Raw confidence value (0.5 to 1.0).

    Returns:
        One of "high", "medium", "low".
    """
    if confidence >= settings.CONFIDENCE_HIGH:
        return "high"
    elif confidence >= settings.CONFIDENCE_MEDIUM:
        return "medium"
    else:
        return "low"
```

Use Pydantic v2 BaseModel with `str | None` union syntax (Python 3.11+, per RESEARCH.md state of the art).

```python
"""Pydantic response models for all API endpoints."""
from pydantic import BaseModel


class PredictionResponse(BaseModel):
    """Single game prediction."""
    game_id: str
    season: int
    week: int
    game_date: str | None = None
    home_team: str
    away_team: str
    predicted_winner: str
    confidence: float
    confidence_tier: str
    actual_winner: str | None = None
    correct: bool | None = None


class HistorySummary(BaseModel):
    """Summary statistics for prediction history."""
    correct: int
    total: int
    accuracy: float | None = None


class PredictionHistoryResponse(BaseModel):
    """Response for GET /predictions/history."""
    predictions: list[PredictionResponse]
    summary: HistorySummary


class WeekPredictionsResponse(BaseModel):
    """Response for GET /predictions/week/{week} and /predictions/current."""
    season: int
    week: int
    status: str = "ok"  # "ok" or "offseason"
    predictions: list[PredictionResponse]


class ModelInfoResponse(BaseModel):
    """Response for GET /model/info."""
    experiment_id: int
    training_date: str
    val_accuracy_2023: float
    feature_count: int
    hypothesis: str
    baseline_always_home: float
    baseline_better_record: float


class ShapFeature(BaseModel):
    """Single SHAP feature importance entry."""
    feature: str
    importance: float


class ExperimentResponse(BaseModel):
    """Single experiment from experiments.jsonl."""
    experiment_id: int
    timestamp: str
    params: dict
    features: list[str]
    val_accuracy_2023: float
    val_accuracy_2022: float
    val_accuracy_2021: float
    baseline_always_home: float
    baseline_better_record: float
    log_loss: float
    brier_score: float
    shap_top5: list[ShapFeature]
    keep: bool
    hypothesis: str
    prev_best_acc: float
    model_path: str | None = None


class ReloadResponse(BaseModel):
    """Response for POST /model/reload."""
    status: str
    experiment_id: int
    val_accuracy_2023: float
    predictions_generated: int


class HealthResponse(BaseModel):
    """Response for GET /api/health."""
    status: str
```

Field names and types:
- PredictionResponse.game_date is `str | None` (serialized from DATE, may be None)
- PredictionResponse.confidence is float (0.5 to 1.0 range)
- PredictionResponse.confidence_tier is str: "high", "medium", or "low"
- HistorySummary.accuracy is `float | None` (None when total=0 to avoid division by zero)
- ModelInfoResponse matches CONTEXT.md: experiment ID, training date, 2023 val accuracy, feature count, hypothesis, AND both baseline accuracies
- ExperimentResponse mirrors the experiments.jsonl entry schema exactly
- ReloadResponse includes predictions_generated count so caller knows what happened
