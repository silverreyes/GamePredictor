# Phase 5 — UI Design Contract

> Visual and interaction contract for the NFL Game Predictor dashboard. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (initialize during plan execution) |
| Preset | Default (New York style) — dark mode, zinc base |
| Component library | Radix UI (via shadcn/ui) |
| Icon library | lucide-react |
| Font | Inter (system fallback: ui-sans-serif, system-ui, sans-serif) |

**Initialization command (run during first plan):**
```
npx shadcn@latest init
```
Select: TypeScript, Tailwind CSS, `src/` directory, CSS variables for colors, zinc base color, dark mode default.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline badge padding |
| sm | 8px | Compact element spacing, table cell padding |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section padding, sidebar item height padding |
| xl | 32px | Layout gaps between major content areas |
| 2xl | 48px | Sidebar width gutters, page top margin |
| 3xl | 64px | Not used in this phase |

Exceptions: Sidebar width fixed at 240px desktop / 180px tablet (not a spacing token — a layout dimension). Minimum card width in picks grid: 280px.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Label | 12px | 400 (regular) | 1.4 | Badges, stat labels, sidebar nav items, confidence tiers |
| Body | 14px | 400 (regular) | 1.5 | Table cells, card body text, descriptions, sidebar title |
| Heading | 20px | 600 (semibold) | 1.2 | Page titles, card section headers, predicted winner name |
| Display | 28px | 600 (semibold) | 1.1 | Hero stat numbers (accuracy percentage, win probability) |

Font stack: `Inter, ui-sans-serif, system-ui, -apple-system, sans-serif`

Monospace for experiment IDs, game IDs, and params: `"JetBrains Mono", ui-monospace, monospace` at 12px weight 400. Monospace uses the Label size tier — font family alone distinguishes code context.

### Page: This Week's Picks (`/`)

| Element | shadcn Component | Data Source |
|---------|-----------------|-------------|
| Page heading | h1 "Week {N} Picks" | `WeekPredictionsResponse.week` |
| Season/week subheading | p "2024 Season" | `WeekPredictionsResponse.season` |
| Picks grid | CSS Grid, 1-3 columns responsive | `WeekPredictionsResponse.predictions[]` |
| Pick card | Card | One per game. Contains: away @ home, predicted winner highlighted, win probability as Display number, confidence tier badge |
| Offseason state | Empty state card | When `status === "offseason"` |

**Pick Card Layout (each game):**
```
+----------------------------------+
| away_team  @  home_team          |  <- 14px regular, team names
|                                  |
|     [predicted_winner]           |  <- 20px semibold, accent color
|        68.2%                     |  <- 28px semibold (Display), win prob
|     [High Confidence]            |  <- Badge with tier color
+----------------------------------+
```
- Predicted winner team name is displayed prominently at 20px semibold
- Non-predicted team displayed at 14px in muted text
- The `@` separator uses muted text
- Card border-left: 3px solid with confidence tier color (blue/amber/zinc)
- If `actual_winner` is set and `correct === true`: green check icon (16px) in top-right corner
- If `actual_winner` is set and `correct === false`: red X icon (16px) in top-right corner

## Loading States

All pages use skeleton loading via shadcn Skeleton component.

| Page | Skeleton Pattern |
|------|-----------------|
| This Week | 3 card-shaped skeletons in grid layout, pulsing |
| Accuracy | 3 summary card skeletons + table skeleton (5 rows) |
| Experiments | Table skeleton with 5 rows, full width |
| History | Filter bar skeleton + table skeleton with 8 rows |
| Sidebar model status | 2-line text skeleton in status card |

Skeleton color: zinc-800 with zinc-700 pulse animation (matches dark theme).

TanStack Query config:
- `staleTime`: 5 minutes (300000ms) for all queries
- `refetchOnWindowFocus`: false
- `retry`: 2

---

## Error States

All errors use the same pattern: centered Card with error icon, heading, body, and retry button.

| Error Type | Heading | Body | Action |
|------------|---------|------|--------|
| Network error | "Connection Failed" | "Could not reach the prediction API. Make sure the server is running on localhost:8000." | Button: "Try Again" (refetches query) |
| 404 (no data) | "No Predictions Found" | "No predictions exist for this week yet. Predictions are generated when the model processes new game data." | None (informational) |
| 500 (server error) | "Something Went Wrong" | "The server encountered an error. Check the API logs for details." | Button: "Try Again" |
| Offseason | "Offseason" | "No upcoming games to predict. Check back when the regular season starts." | Link: "View History" (navigates to /history) |

Error icon: AlertCircle from lucide-react, 32px, red-500.
"Try Again" button: Button variant="outline", size="sm".

## Responsive Breakpoints

| Breakpoint | Width | Layout Change |
|------------|-------|---------------|
| Desktop (default) | >= 1024px | Sidebar 240px + content area. Picks grid 3 columns. |
| Tablet | 768px - 1023px | Sidebar narrows to 180px with smaller labels (icon + label always visible). Picks grid 2 columns. |
| Mobile | < 768px | Sidebar becomes top nav bar (horizontal, icons + labels in a row). Picks grid 1 column. Tables scroll horizontally. |

Content area max-width: 1200px, centered with auto margins.

Sidebar always shows icon + label at every breakpoint (per CONTEXT.md locked decision: "Always-visible sidebar with icon + label for each nav item (no collapse)"). At tablet width the sidebar uses 180px width with 12px label text and tighter 4px icon-to-label gap to fit the narrower container.

## Interaction Contracts

### Navigation
- Sidebar nav items are `NavLink` components from React Router
- Active route: blue-500/10 background + blue-400 text color
- Hover (non-active): zinc-800 background
- Transition: 150ms ease background-color

### Tables
- Row hover: zinc-800/50 background
- Sortable column headers: cursor pointer, ChevronUp/ChevronDown icon indicating sort direction
- Default sort: experiments by experiment_id descending, history by week ascending

### Cards (picks)
- No click interaction (cards are display-only)
- Border-left color indicates confidence tier
- Subtle hover: translateY(-1px) with 150ms transition (provides visual feedback without implying clickability)

### Filters (history page)
- Season Select: populated from available seasons in data
- Team Select: "All Teams" default + alphabetical team list
- Filters update URL query params for shareability
- TanStack Query re-fetches on filter change
