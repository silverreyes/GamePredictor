## Tech Debt

### Phase 2: Feature Engineering

- `SITUATIONAL_FEATURES` constant in `features/definitions.py` lists stale column names (`is_home`, `home_rest_days`, `away_rest_days`) that don't match actual DataFrame output (`home_rest`, `away_rest`, `div_game`). The constant is never imported by any runtime code path — documentation-only risk for future developers.

### Phase 4: Prediction API

- `CORS_ORIGINS` in `api/config.py` is hardcoded to `["http://localhost:3000", "http://localhost:5173"]` and not configurable via environment variable. Does **not** break the primary production deployment behind Caddy (SPA uses relative URLs → same-origin requests). Only affects cross-origin API consumers or direct API access from non-localhost origins.

### Phase 6: Pipeline & Deployment

- First-run model seeding is not documented. On fresh `docker compose up`, the `models` named volume is empty. The API starts gracefully with `model=None` (returns HTTP 503 for model endpoints), and dashboard pages show empty states. A manual step to copy `models/artifacts/best_model.json` and `models/experiments.jsonl` into the volume (or calling the worker manually) is required but not documented in `.env.example` or any deployment guide.
