## Interaction Contract

### PickCard Spread Display

**Pre-game state (no actual_winner):**
```
[existing card content]
  67.2%                          <- confidence (text-[28px])
  High                           <- tier badge
  +7.5 spread                    <- NEW: text-sm text-muted-foreground
```

**Post-game state (actual_winner present, spread data available):**
```
[existing card content]
  67.2%                          <- confidence (text-[28px])
  High                           <- tier badge
  +7.5 spread                    <- spread prediction
  Actual +10 (off by 2.5)        <- NEW: text-xs, "off by" number color-coded
```

**No spread data (model not loaded or no spread predictions):**
```
[existing card content unchanged -- no spread lines rendered]
```

Decision: SpreadLabel component receives `predictedSpread: number | undefined` and `actualSpread: number | undefined`. When `predictedSpread` is undefined, renders nothing (null return). This ensures graceful degradation to v1.0 appearance.

### Spread Error Color Coding

| Condition | Color Class | Example |
|-----------|-------------|---------|
| `Math.abs(predictedSpread - actualSpread) <= 3` | `text-green-400` | "off by 2.5" |
| `Math.abs(predictedSpread - actualSpread) > 3 && <= 7` | `text-amber-400` | "off by 5.3" |
| `Math.abs(predictedSpread - actualSpread) > 7` | `text-red-400` | "off by 12.1" |

### Accuracy Page Spread Section

**Layout (below classifier SummaryCards and Week-by-Week table):**
```
[existing classifier SummaryCards -- untouched]
[existing Week-by-Week Breakdown table -- untouched]

Spread Model                     <- section heading (text-sm font-semibold uppercase tracking-wide)
  [Spread MAE card]  [Winner Accuracy card]  [Agreement card]
```

Three-card flex row using identical layout pattern as classifier SummaryCards: `flex flex-col md:flex-row gap-6`, each card `flex-1`.

**Card 1: Spread MAE**
```
  SPREAD MAE                     <- text-sm text-muted-foreground uppercase tracking-wide mb-2
  10.68                          <- text-[28px] font-semibold leading-tight
  points average error           <- text-sm text-muted-foreground mt-1
```

**Card 2: Winner Accuracy**
```
  SPREAD WINNER ACCURACY         <- label
  60.2%                          <- stat display
  [Beating/Behind badge]         <- ComparisonBadge comparing spread vs classifier
```

The ComparisonBadge compares `spread_winner_accuracy` against `classifier_accuracy`. This reuses the existing `ComparisonBadge` component already in `SummaryCards.tsx` (extract to shared if needed, or inline the same pattern).

**Card 3: Agreement Breakdown**
```
  CLASSIFIER VS SPREAD           <- label
  Both correct: 142              <- text-sm, each on its own line
  Both wrong: 38
  Only classifier: 15
  Only spread: 10
```

This card does not use the display stat pattern. It uses a compact list format since it shows 4 values. Each line is `text-sm` with the count in `font-semibold`.

### Conditional Visibility

| Condition | Behavior |
|-----------|----------|
| `spread_model` is null in ModelInfoResponse | Hide entire "Spread Model" section on Accuracy page |
| Spread predictions API returns empty array or 404 | Hide spread lines on PickCards (SpreadLabel renders null) |
| Spread predictions API is loading | Show skeleton for spread section on Accuracy page; PickCards render without spread lines (no skeleton on individual cards) |

### Loading States

**Accuracy page spread section skeleton:**
```
[Spread Model heading skeleton: h-4 w-32]
[Three card skeletons: h-32 flex-1 rounded-lg] -- matches existing classifier skeleton pattern
```

**ThisWeekPage / PickCards:** No spread-specific loading skeleton. Spread lines appear once spread data arrives. If spread data loads after classifier data, cards initially render without spread lines and update when spread data arrives. This avoids layout shift anxiety -- the spread line is small (one line of text-sm) and its appearance is not jarring.

### Data Fetching Strategy

**ThisWeekPage:** Two parallel `useQuery` calls -- `useCurrentPredictions()` (existing) and `useSpreadPredictions(season, week)` (new). The page renders PickCards as soon as classifier data arrives. Spread data is joined client-side by `game_id` and passed to PicksGrid/PickCard. When spread query is still loading or errored, PickCards render without spread lines.

**AccuracyPage:** `useModelInfo()` already returns `spread_model: SpreadModelInfo | null` (MAE, derived_win_accuracy). For the agreement breakdown, fetch both `usePredictionHistory()` (existing) and a new spread history call. Agreement counts are computed client-side by joining on `game_id` and comparing `correct` fields. Only games where both predictions have `correct !== null` are included.

### History Page

If the History page (`HistoryTable.tsx`) shows individual game rows, add a `Spread` column showing the predicted spread for each game. This follows the same conditional pattern: if spread data is unavailable, the column is omitted entirely. Implementation is at Claude's discretion during planning -- this is lower priority than PickCard and Accuracy page integration.

| Element | Copy |

| Spread section heading (Accuracy) | "Spread Model" |
| Spread MAE card label | "Spread MAE" |
| Spread MAE card subtitle | "points average error" |
| Winner Accuracy card label | "Spread Winner Accuracy" |
| Agreement card label | "Classifier vs Spread" |
| Agreement line: both correct | "Both correct: {n}" |
| Agreement line: both wrong | "Both wrong: {n}" |
| Agreement line: only classifier | "Only classifier: {n}" |
| Agreement line: only spread | "Only spread: {n}" |
| Empty state (no spread model) | Section hidden entirely -- no copy needed (CONTEXT.md decision) |
| Empty state (no spread predictions) | Spread lines hidden on PickCards -- no copy needed (CONTEXT.md decision) |
| Error state (spread API fails) | No dedicated error -- spread section silently absent, classifier data still shows (graceful degradation) |
| Destructive actions | None in this phase |

### Number Formatting

| Value | Format | Example |
|-------|--------|---------|
| Predicted spread | 1 decimal, signed, "spread" suffix | "+7.5 spread", "-3.2 spread", "+0.0 spread" |
| Actual spread | 1 decimal, signed | "+10.0" |
| Spread error | 1 decimal, unsigned | "2.5" |
| Spread MAE | 2 decimals | "10.68" |
| Winner accuracy | 1 decimal + "%" | "60.2%" |
| Agreement counts | Integer | "142" |

### Sign Convention

Positive spread = home team favored (home team predicted to win by that many points).
Negative spread = away team favored.
Zero spread = "+0.0 spread" (displayed with positive sign per home-team convention from Phase 8: predicted_spread == 0.0 means home wins).
