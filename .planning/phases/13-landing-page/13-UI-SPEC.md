## Gradient Treatment

Hero section background includes a subtle radial amber glow behind the accuracy stat:

```
bg-radial-[at_50%_65%] from-primary/8 to-transparent
```

- Position: centered horizontally, 65% from top (behind the stat area)
- Start color: amber primary at 8% opacity
- End color: transparent
- Applied to an absolutely-positioned overlay `div` with `pointer-events-none`
- Uses Tailwind v4 `bg-radial` syntax (NOT the deprecated TW3 `bg-gradient-to-*`)

Source: CONTEXT.md "subtle gradient background (dark-to-slightly-lighter or faint amber radial glow behind the stat)", Claude's discretion for exact opacity and positioning.

## Responsive Behavior

| Breakpoint | Behavior |
|------------|----------|
| Mobile (<768px / default) | Hero `min-h-[50vh]`, headline `text-4xl`, stat `text-5xl`, subtitle `text-base`, How It Works `grid-cols-1`, all content `px-6` |

## Anti-Patterns (Do NOT)

1. **Do NOT use hex, rgb, or raw oklch values in className** -- always use semantic token classes (`text-primary`, `bg-background`, etc.)
2. **Do NOT use the Card component** for How It Works blocks -- use plain `div` with `bg-secondary/50 rounded-lg p-6`
3. **Do NOT use TW3 gradient syntax** (`bg-gradient-to-*`) -- use TW4 `bg-linear-to-*` or `bg-radial`
4. **Do NOT use `<Button asChild>`** -- use `buttonVariants()` applied directly to `<Link>` elements
5. **Do NOT import `hero.png`** as a banner image substitute -- it is a Vite starter icon
6. **Do NOT create separate sub-components** for each section -- single-file `LandingPage.tsx` with commented sections
7. **Do NOT use `window.location`** for navigation -- use react-router `Link` component
8. **Do NOT put footer in `LandingLayout.tsx`** -- keep everything in `LandingPage.tsx` for simplicity (only one page uses this layout)

Source: RESEARCH.md Anti-Patterns, RESEARCH.md Common Pitfalls.
