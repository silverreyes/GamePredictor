class PredictionResponse(BaseModel):
    game_id: str

    training_date: str
    val_accuracy_2023: float
    feature_count: int
    hypothesis: str
    baseline_always_home: float
    baseline_better_record: float

class ShapFeature(BaseModel):
    feature: str
    importance: float

    experiment_id: int
    timestamp: str
    params: dict
    features: list[str]

    val_accuracy_2022: float
    val_accuracy_2021: float

**B. API client** — Create `frontend/src/lib/api.ts`:
- `const API_BASE = import.meta.env.VITE_API_URL ?? "http://localhost:8000"` — configurable base URL with localhost fallback.
- Generic `apiFetch<T>(path: string): Promise<T>` that does `fetch(API_BASE + path)`, checks `res.ok`, throws `Error("API error: {status} {statusText}")` on failure, returns `res.json()`.
- Export 5 named functions:
  - `fetchCurrentPredictions()` -> GET `/api/predictions/current` -> `WeekPredictionsResponse`
  - `fetchWeekPredictions(week: number, season?: number)` -> GET `/api/predictions/week/{week}?season={season}` -> `WeekPredictionsResponse`
  - `fetchPredictionHistory(season?: number, team?: string)` -> GET `/api/predictions/history` with optional query params using `URLSearchParams` -> `PredictionHistoryResponse`
  - `fetchModelInfo()` -> GET `/api/model/info` -> `ModelInfoResponse`
  - `fetchExperiments()` -> GET `/api/experiments` -> `ExperimentResponse[]`

**D. Custom hooks** — Create 4 hooks in `frontend/src/hooks/`:

- `useCurrentPredictions.ts`: wraps `useQuery<WeekPredictionsResponse>` with queryKey `["predictions", "current"]` and queryFn `fetchCurrentPredictions`.
- `useModelInfo.ts`: wraps `useQuery<ModelInfoResponse>` with queryKey `["model", "info"]` and queryFn `fetchModelInfo`.
- `useExperiments.ts`: wraps `useQuery<ExperimentResponse[]>` with queryKey `["experiments"]` and queryFn `fetchExperiments`.
- `usePredictionHistory.ts`: accepts optional `season?: number` and `team?: string` params. Uses queryKey `["predictions", "history", { season, team }]` and queryFn `() => fetchPredictionHistory(season, team)`.

All hooks import from `@tanstack/react-query` and `@/lib/api`. None use `onSuccess`/`onError` callbacks (removed in TanStack Query v5).
