## Typography

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Body | IBM Plex Mono | 14px | 400 (regular) | 1.5 | Default body text, data values, navigation items, badges |
| Label | IBM Plex Mono | 12px | 400 (regular) | 1.4 | Table headers, stat labels, muted captions |
| Heading | Syne | 20px | 700 (bold) | 1.2 | Page titles (h2, h3), section headings |
| Display | Syne | 28px | 700 (bold) | 1.1 | Hero text (Phase 13), primary h1 headings |

**Weight consolidation rationale:** Two weights provide sufficient hierarchy. Labels at 12px are already visually distinct from 14px body text through size alone -- adding a third weight (500) would be unnecessary. Display headings at 28px are already visually distinct from 20px headings through size alone -- weight 700 provides strong emphasis for both without requiring an 800 weight. This keeps font loading lean (fewer weight files) and avoids visual clutter from too many weight variations.

### Font Loading

| Package | Weights Loaded | Why |
|---------|---------------|-----|
| @fontsource/syne | 400, 700 | 400 fallback body, 700 headings + display |
| @fontsource/ibm-plex-mono | 400, 600 | 400 body + labels, 600 reserved for future bold mono emphasis if needed |

Note: Only weights 400 and 700 are actively used in the type scale. IBM Plex Mono 600 is loaded as a safety margin but not assigned to any role. Syne 400 is loaded as a fallback for any edge case where Syne renders outside a heading context.

### Font Assignment Rules

- `--font-display: "Syne", sans-serif` applies to: h1, h2, h3, page titles, "NFL Predictor" brand text
- `--font-sans: "IBM Plex Mono", ui-monospace, monospace` applies to: body text, labels, code, data values, navigation items, badges (this is the DEFAULT font)
- `--font-mono: "IBM Plex Mono", ui-monospace, monospace` aliases the same font (body IS monospace, matching silverreyes.net)
- Base layer rule: `h1, h2, h3 { font-family: var(--font-display); }` provides automatic heading styling
- Tailwind class `font-display` available for manual override on any element needing Syne

### Font Migration

| Before | After |
|--------|-------|
| Google Fonts CDN (`@import url('https://fonts.googleapis.com/...')`) | @fontsource packages (self-hosted, bundled) |
| Inter (body) | IBM Plex Mono (body) |
| JetBrains Mono (code) | IBM Plex Mono (code -- same font as body) |

### Accent Reserved For

The amber accent (oklch(0.767 0.157 71.7)) is restricted to these specific elements:

1. **Active navigation item** text color (sidebar)
2. **Primary buttons** (background fill) -- used sparingly, mainly Phase 13 CTAs
3. **Focus ring** on interactive elements
4. **Accuracy stat** number on landing page (Phase 13)
5. **"Kept" experiment** indicator border/accent (Phase 14)
6. **Progress bar** fill in ExperimentDetail
7. **Link hover** state (replacing blue-400)

Accent must NOT be used for: body text, headings, card backgrounds, borders (use --border token), or generic UI chrome.
