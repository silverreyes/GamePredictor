**1. Create `Dockerfile`** (multi-stage Python-only build):

Per CONTEXT.md, Caddy serves `frontend/dist/` via bind mount (`./frontend/dist:/srv` in docker-compose). The frontend is pre-built on the host (`npm run build` during VPS setup). The Dockerfile only needs to package the Python application.

```dockerfile
FROM python:3.11-slim AS builder
WORKDIR /app
COPY pyproject.toml ./
RUN pip install --no-cache-dir --prefix=/install .

FROM python:3.11-slim AS runtime
WORKDIR /app
COPY --from=builder /install /usr/local
COPY . .
ENV PYTHONUNBUFFERED=1
ENV PYTHONDONTWRITEBYTECODE=1
```

Do NOT add a `CMD` directive -- the entrypoint is set per-service in docker-compose.yml (uvicorn for api, python -m pipeline.worker for worker).

**2. Create `mlflow.Dockerfile`:**

```dockerfile
FROM ghcr.io/mlflow/mlflow:latest
RUN pip install --no-cache-dir psycopg2-binary
```

This extends the official MLflow image with the PostgreSQL driver it lacks (confirmed by RESEARCH.md Pitfall 1).

**3. Create `Caddyfile`:**

```
{$DOMAIN:localhost} {
    encode gzip

    handle /api/* {
        reverse_proxy api:8000
    }

    handle {
        root * /srv
        try_files {path} /index.html
        file_server
    }
}
```

The `{$DOMAIN:localhost}` syntax reads the DOMAIN env var, defaulting to localhost for local dev. `try_files {path} /index.html` handles SPA client-side routing.

**4. Create `.env.example`:**

- `postgres`: mounts `./sql` directory (not single file) so both `00-create-mlflow-db.sh` and `init.sql` run alphabetically
- `api`: models volume mounted `:ro` (read-only) -- API reads model artifacts, never writes
- `worker`: models volume mounted read-write (no `:ro`) -- worker stages candidate models
- `mlflow`: NO ports mapping (internal-only per CONTEXT.md). Access via SSH tunnel when needed.
- `caddy`: depends on api healthy, serves `./frontend/dist` at `/srv`
- All database URLs use `${POSTGRES_PASSWORD:-nfldev}` with dev fallback

**2. Create `sql/00-create-mlflow-db.sh`:**

This shell script runs BEFORE `init.sql` during postgres initialization (files in docker-entrypoint-initdb.d run alphabetically: `00-create-mlflow-db.sh` < `init.sql`). `CREATE DATABASE` is a cluster-level command that cannot run inside a `.sql` file connected to a specific database.

```bash
#!/bin/bash
set -e
psql -v ON_ERROR_STOP=1 --username "$POSTGRES_USER" --dbname "$POSTGRES_DB" <<-EOSQL
    CREATE DATABASE mlflow OWNER $POSTGRES_USER;
EOSQL
```

Make the script executable: `chmod +x sql/00-create-mlflow-db.sh` (or set executable bit in git).

Do NOT modify `sql/init.sql` -- it continues to create tables in the nflpredictor database as before.
