## Interaction Contract

### Navigation Active State

| Route | Active Nav Item | Layout |
|-------|----------------|--------|
| `/` | Home | LandingLayout (full-width, no sidebar) |
| `/this-week` | This Week | AppLayout (sidebar + constrained content) |
| `/accuracy` | Accuracy | AppLayout |
| `/experiments` | Experiments | AppLayout |
| `/history` | History | AppLayout |

**Critical detail:** The Home nav item uses `NavLink` with the `end` prop so it only shows active state when the URL is exactly `/`, not on every page.

### Layout Transition Behavior

| From | To | Transition |
|------|-----|-----------|
| Landing page (`/`) | Dashboard (`/this-week`) | Instant -- no animation, no fade. Full page layout swap (landing has no sidebar; dashboard has sidebar). |
| Dashboard (any) | Landing page (`/`) | Instant -- no animation. Sidebar disappears, full-width layout renders. |
| Dashboard (any) | Dashboard (other) | Instant -- existing behavior. Sidebar persists, only content area updates. |

Rationale: CSS transitions between fundamentally different layouts (full-width vs sidebar) add complexity for minimal UX value. Clean, instant swaps are standard for this application type.

### Placeholder Landing Page Interaction

The placeholder page at `/` is a temporary bridge between Phase 12 (route setup) and Phase 13 (landing content).

| Element | Behavior |
|---------|----------|
| "NFL Nostradamus" heading | Static text, centered, Syne display font |
| "Landing page coming soon." | Static text, centered, muted foreground |
| "Go to Dashboard" button | Navigates to `/this-week` via React Router `<Link>`. Uses `bg-primary` amber fill. Hover state: `bg-primary/90`. |

### LandingLayout Visual Contract

| Property | Value | Rationale |
|----------|-------|-----------|
| Width | Full viewport (no max-width constraint) | Standalone layout for landing; Phase 13 controls its own content width |
| Height | `min-h-screen` | Ensures background covers full viewport |
| Background | `bg-background` (oklch(0.134 0.003 106.7)) | Same near-black as dashboard for visual consistency |
| Text | `text-foreground` (oklch(0.883 0.015 84.6)) | Same warm text as dashboard |
| Sidebar | None | Defining characteristic -- this layout has NO sidebar |
| Mobile nav bar | None | Landing page has no navigation chrome |
| Content wrapper | `<main>` with `<Outlet />` inside | Clean semantic HTML; Phase 13 controls all inner styling |
